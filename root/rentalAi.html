🔭 Vision & impact

renting.ai is an early-stage, production-minded MVP that transforms rental discovery through Retrieval-Augmented Generation (RAG) and semantic search. Instead of rigid filters, users type “2 BHK with parking near mandi under ₹12k” and get hyper-relevant results + AI-generated summaries.

current focus Lakhimpur Kheri (UP's largest district) · Lucknow · expanding to Delhi-NCR, Mumbai, Bengaluru in 2026.

⚡ Hybrid RAG — vector similarity (Qdrant) + structured filters (SQLite) + LLM summary (OpenAI).
🧩 Full-stack GenAI — Sentence embeddings, 384-dim, cosine search, LangChain prompt chaining.
📱 Mobile-first UX — lightweight HTML/CSS, fast on 2G, built for semi‑urban India.

💻 1. core programming & frontend

  • Python 3.10+ – modular OOP, ETL scripts (csvToJson.py), async FastAPI, batch embedding logic. Error handling, generator patterns for 1000+ listings.
  • HTML5 + Jinja2 – semantic templates (search form, property cards) with dynamic rendering. Mobile‑optimised for low bandwidth.
  • CSS3 (flex/grid) – fully responsive, card‑based UI, 27% repo weight shows frontend depth. Dark/light ready.
  • JavaScript ES6 – client‑side validation, loading states, async fetch for preview, no heavy frameworks → lightweight MVP.

🧠 2. AI/ML · RAG · semantic core

  • Vector embeddings (Sentence‑Transformers) – 384‑dim dense vectors from search_text; enables meaning‑based matching (“parking ke paas flat” → relevant). Batch upsert in saveEmbeddings.py.
  • Hybrid RAG (retrieval‑augmented generation) – retrieval: Qdrant (cosine) + SQLite filters (rent, bedrooms). Generation: OpenAI GPT via LangChain prompt → natural answer + top listings. Drastically reduces hallucination.
  • Semantic search (cosine similarity) – Qdrant Distance.COSINE, returns top‑k semantically similar properties. Handles Hinglish and vernacular terms.
  • LLM integration (OpenAI) – environment‑key protected, prompt templates in contextGenerations/prompt.py / LangChain for concise, user‑friendly summaries.

🗄️ 3. database & data engineering

  • Vector DB (Qdrant) – local instance, collection with 384‑dim params, batch upsert (size 100) with payload (ad_id, text, rent). Sub‑second queries.
  • Relational (SQLite) – schema with 30+ fields (sqlliteDatabase.py), indexes on city/rent/bedrooms, to_sql bulk insert.
  • ETL pipeline – csvToJson.py reads raw CSV, applies create_search_text() enrichment (template string), exports JSON, then loads into SQLite & Qdrant.
  • Batch processing & optimisation – memory‑safe loops, idempotent inserts, handling of large future datasets.

🔌 4. backend & API architecture

  • FastAPI (async) – high‑performance endpoints (/search, /health), automatic OpenAPI docs, Pydantic validation, Jinja2 templating.
  • Modular services – query validation, embedding dispatcher, vector search, RAG orchestration (inferred queryEmbeddingService/, langchainRag/).
  • RESTful design & input sanitisation – safe handling of natural language, guard against injection, clean parameter extraction.
  • Error handling & idempotency – try/catch in scripts, IF NOT EXISTS in SQLite, graceful fallback if LLM fails.

☁️ 5. deployment & devops mindset

  • Uvicorn + local dev – --reload for fast iteration, ready to wrap in Docker/Gunicorn for production.
  • Modular architecture – separation of ingestion, embedding, web, tests; easy substitution (e.g. swap Qdrant cloud).
  • Environment security – .env for OPENAI_API_KEY, .gitignore protects secrets & DB, pinned requirements.txt.
  • CI/CD ready – GitHub Actions workflows (ci.yml , cd.yml) lint, test, docker build. Deployment via tags.

⚡ quick start (dev)

git clone https://github.com/shaanMS/renting.ai
cd renting.ai
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # add your OPENAI_API_KEY

# data pipeline
python contextGenerations/csvToJson.py
python database/sqlliteDatabase.py
python vectorsProcessing/saveEmbeddings.py

# run app
uvicorn webApp.main:app --reload
# open http://localhost:8000

🐳 Docker also supported – see docker-compose.yml.
